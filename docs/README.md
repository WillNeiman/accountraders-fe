## 문서 작성 가이드라인

1. 모든 문서는 마크다운(.md) 형식으로 작성합니다.
2. 문서는 가능한 한 간결하고 명확하게 작성합니다.
3. 코드 예시는 가능한 한 실제 프로젝트의 코드를 사용합니다.
4. 문서 간 중복되는 내용은 참조를 통해 연결합니다.

## 문서 업데이트

문서를 업데이트할 때는 다음 사항을 고려해주세요:

1. 변경 사항의 명확한 설명
2. 변경 이유
3. 영향 받는 다른 문서들
4. 마이그레이션 가이드 (필요한 경우)
